Arch Linux Installation Commands

 english  software      2021-03-06     290 words      2 minutes
Contents

I’m learning about Arch Linux so I can install it on a custom build desktop.

I’ll do that by doing an installation using VirtualBox.

It took me eight attempts to stabilize the script below.

Download ISO, e.g., from https://ftp.rnl.tecnico.ulisboa.pt/pub/archlinux/iso/2021.03.01.

Create an image and activate EFI in the motherboard settings.

Start the virtual machine and follow the instructions.

The following instructions assume Portugal as the target locale.

Activate the Portuguese keyboard layout, check if networking is working, and update the system clock:

loadkeys pt-latin1 ip link ping archlinux.org timedatectl set-ntp true

Next, create the partitions using fdisk:

fdisk -l fdisk /dev/sda n default default default +512M t L (check for EFI) ef n default default default +1024M t default L 82 (check for linux swap) n default default default default w

After the partition creation:

mkfs.fat -F32 /dev/sda1 mkswap /dev/sda2 swapon /dev/sda2 mkfs.ext4 /dev/sda3 mount /dev/sda3 /mnt pacstrap /mnt base linux linux-firmware vim genfstab -U /mnt » /mnt/etc/fstab

With the base setup done, let’s set up the locale:

arch-chroot /mnt ln -sf /usr/share/zoneinfo/Europe/Lisbon /etc/localtime hwclock –systohc vi /etc/locale.gen

uncomment pt_PT and en_GB ones

:x locale-gen

Now set the network:

vi /etc/hostname arch5 (the name you want) :x vi /etc/hosts 127.0.0.1 localhost ::1 localhost 127.0.0.1 arch5.localdomain arch5 :x pacman -S networkmanager gnome-keyring systemctl enable NetworkManager.service

Change the root password:

passwd

And configure the bootloader:

pacman -S grub efibootmgr mkdir /boot/efi mount /dev/sda1 /boot/efi grub-install –target=x86_64-efi –bootloader-id=GRUB –efi-directory=/boot/efi grub-mkconfig -o /boot/grub/grub.cfg

To finish installing the desktop environment:

pacman -S xorg pacman -S gnome-shell nautilus gnome-terminal guake gnome-tweak-tool gnome-control-center xdg-user-dirs gdm systemctl enable gdm.service exit reboot

The network was not working properly in one of the trials.

vi /etc/systemd/network/20-wired.network [Match] Name=enp0s3 (name of the network here)

[Network] DHCP=yes :x systemctl restart systemd-networkd.service ping google.com
